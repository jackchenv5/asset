# 环境配置说明

## 文件结构

本项目使用以下环境变量文件：

- `.env` - 公共环境变量（所有环境共享）
- `.env.development` - 开发环境变量
- `.env.production` - 生产环境变量
- `.env.example` - 环境变量配置示例

## 配置说明

### 1. 公共环境变量 (.env)
```bash
# 应用基本信息
VITE_APP_NAME=资产管理系统
VITE_APP_VERSION=1.0.0
```

### 2. 开发环境 (.env.development)
```bash
# 开发环境API配置
VITE_API_BASE_URL=http://localhost:8002
VITE_API_PREFIX=/api
```

### 3. 生产环境 (.env.production)
```bash
# 生产环境API配置
VITE_API_BASE_URL=http://your-production-domain.com
VITE_API_PREFIX=/api
```

## 使用方式

### 开发环境运行
```bash
npm run dev
# 或
pnpm dev
```

### 构建命令
```bash
# 默认构建（使用生产环境配置）
npm run build

# 开发环境构建
npm run build:dev

# 生产环境构建
npm run build:prod
```

## 环境变量使用

在代码中使用环境变量：
```javascript
// 获取API基础地址
const baseUrl = import.meta.env.VITE_API_BASE_URL

// 获取API前缀
const apiPrefix = import.meta.env.VITE_API_PREFIX

// 获取应用名称
const appName = import.meta.env.VITE_APP_NAME
```

## 注意事项

1. 环境变量必须以 `VITE_` 开头才能在客户端代码中访问
2. 修改环境变量后需要重启开发服务器
3. `.env.production` 中的配置应该指向你的实际生产环境地址
4. 不要将敏感信息（如密码、密钥）放在环境变量文件中

## 部署配置

在部署到生产环境时，请确保：

1. 修改 `.env.production` 中的 `VITE_API_BASE_URL` 为你的实际域名
2. 使用 `npm run build:prod` 进行构建
3. 将构建产物部署到你的Web服务器
